# 텍스터브 콘텐츠 운영 가이드

지속 가능한 콘텐츠 발행을 위한 **템플릿·캘린더·품질 게이트** 모음.

---

## 1. 발행 리듬 & 캘린더

### 1.1 주간 리듬 (목표: 월 15~20편)
| 요일 | 콘텐츠 유형 | 비고 |
|---|---|---|
| 월 | 블로그 (도구 사용법) | 신규 도구·업데이트 알림 겸용 |
| 수 | 블로그 (상황별 가이드) | 검색 의도 강한 롱테일 |
| 금 | 블로그 (비교/케이스/통계) | 백링크·공유 유도 |
| 격주 토 | 가이드 (롱폼) | 분량 4,000~6,000자 |
| 매일 | AI 자동 글 | DB 큐(`/blog/ai/[slug]`)에서 1편 자동 공개 |

### 1.2 분기별 작업
- **상위 10개 글 갱신** — 통계·날짜·내부 링크 점검
- **GSC Top Queries 리뷰** → SEO-KEYWORD-MAP.md 보강
- **카니발리제이션 점검** (제목·H1 중복)
- **죽은 링크(404) 체크**

### 1.3 신규 콘텐츠 발행 절차
1. SEO-KEYWORD-MAP에서 키워드 선정 (중복 X)
2. 템플릿 복제 → 작성
3. 품질 게이트 체크리스트 통과
4. `lib/blog-posts-batch{N}.ts` 또는 `lib/guides-registry.ts`에 추가
5. PR → 자동 빌드 → 배포 → GSC 색인 요청
6. SEO-KEYWORD-MAP에 신규 항목 표시

## 2. 블로그 포스트 템플릿

```typescript
{
  slug: "kebab-case-slug",
  title: "[메인 키워드] 포함 매력적인 제목 (50자 이내)",
  description: "검색결과 미리보기에 노출될 1~2문장 요약 (155자 이내). [메인 키워드]를 자연스럽게 포함.",
  content: `# 제목 (검색의도 정확히 매칭)

도입부 2~3문장. 독자의 문제·고민을 즉시 짚고, 글에서 얻을 것을 명시.

## 핵심 결론 (TL;DR)

- 답 1
- 답 2
- 답 3

## H2 섹션 1: 배경/이유

본문…

### H3: 세부 항목

본문…

## H2 섹션 2: 방법/단계

1. 단계
2. 단계
3. 단계

## H2 섹션 3: 실전 예시 또는 비교표

| 항목 | A | B |
|---|---|---|
| 예 | … | … |

## H2 섹션 4: 자주 하는 실수

- 실수 1
- 실수 2

## 자주 묻는 질문 (FAQ)

(별도 blog-faq-registry에 등록 권장)

## 결론

요약 + 다음 행동(CTA: [관련 도구](/tools/slug)).`,
  publishedAt: "YYYY-MM-DD",
  metaTitle: "[메인 키워드] - [부가 정보] | 텍스터브",  // 60자 이내
  metaDescription: "[메인 키워드]에 대해 [구체적 가치]. [보조 키워드 1~2개] 자연스럽게 포함.",  // 155자 이내
  targetKeywords: ["메인", "보조1", "보조2", "롱테일1", "롱테일2"],  // 3~7개
  category: "가이드 | 정보 | 분석 | 비교",
  tags: ["주제", "관련태그"],
  cta: { text: "지금 [도구명] 사용해보기", link: "/tools/slug" },
  inlinks: [   // 최소 2개 — 관련 글로 트래픽 순환
    { text: "관련 가이드 제목", link: "/blog/related-slug" },
  ],
  outlinks: [  // 1~3개 — 권위 있는 외부 자료
    { text: "외부 자료 제목", link: "https://…", description: "한 줄 설명" },
  ],
  aeoQuestion: "독자가 검색창에 입력할 법한 질문",  // GEO/AEO 최적화
  aeoAnswer: "30~80자의 명확한 한 줄 답변",
}
```

## 3. 가이드 (롱폼) 템플릿

```typescript
{
  slug: "kebab-case-guide-slug",
  title: "[목적] [방법]을 한 번에 정리하는 법",
  description: "이 가이드를 통해 얻을 결과 1문장 + 보조 설명 1문장.",
  relatedWorkflowId: "workflow-slug",  // 선택
  sections: [
    {
      title: "왜 [주제]가 필요한가요?",
      content: "배경 + 독자 문제 정의. 800~1,200자.",
    },
    {
      title: "1단계: [작업명]",
      content: "구체적 절차 + 도구 사용법 + 팁. 1,000~1,500자.",
      relatedToolId: "tool-slug",
    },
    // 2~4단계 동일
    {
      title: "체크리스트와 자주 하는 실수",
      content: "마무리 점검 + 실수 방지. 800~1,200자.",
    },
  ],
```

## 4. 콘텐츠 품질 게이트 (PR 머지 전 체크)

### 4.1 필수 (모두 ✅)
- [ ] 제목에 메인 키워드 포함
- [ ] 메타 디스크립션 155자 이내, 메인 키워드 1회 이상
- [ ] 본문 2,500자 이상 (도구형은 1,500자 허용)
- [ ] H2 섹션 4개 이상
- [ ] 내부 링크 3개 이상 (도구 1+ / 가이드 1+ / 블로그 1+)
- [ ] 외부 권위 링크 1개 이상 (rel="noopener noreferrer")
- [ ] FAQ 등록 (blog-faq-registry 또는 guide-faq-registry)
- [ ] CTA 명확 (관련 도구 페이지 링크)
- [ ] 동적 OG 이미지 자동 생성 확인
- [ ] aeoQuestion / aeoAnswer 작성

### 4.2 권장
- [ ] 표(table) 또는 코드블록 1개 이상
- [ ] 단계별 번호 리스트
- [ ] 수치·통계 인용 (출처 명시)
- [ ] 실제 사례 또는 스크린샷 비유
- [ ] 모바일에서 읽기 좋은 짧은 문단 (3~4줄)

### 4.3 AI 자동 글의 추가 게이트
- [ ] 사람 검수 1회 (오탈자·사실관계)
- [ ] 본문 2,500자 이상 (자동 거절)
- [ ] 내부 링크 3개 이상 자동 삽입
- [ ] FAQ 3개 이상 자동 생성

## 5. 발행 후 모니터링

### 5.1 측정 지표 (주간)
- GSC: 노출수, 클릭수, CTR, 평균 순위
- GA4: 페이지뷰, 평균 체류시간, 이탈률
- Vercel Speed Insights: LCP/INP/CLS
- AdSense: RPM, 페이지 RPM

### 5.2 액션 임계치
| 지표 | 임계치 | 액션 |
|---|---|---|
| CTR < 2% (4주 평균) | 메타·OG 재작성 |
| 평균 순위 11~20위 | 본문 보강·내부 링크 추가 |
| 체류시간 < 30초 | 도입부 재작성·요약 강화 |
| 이탈률 > 90% | 관련 글 위젯·CTA 재배치 |
